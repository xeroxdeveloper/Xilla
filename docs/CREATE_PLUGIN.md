# Система Плагинов (Plugins) в Xilla

*Добавлено в Alpaca B9*

Помимо сложной системы Пакетов (Packages) и стандартных Модулей, в Xilla появилась невероятно простая система **Плагинов**. 

Плагин — это обычный Python-файл, который кладется в папку `xilla/plugins/`. В отличие от встроенных модулей (которые обновляются через `.update`), файлы в папке `plugins` полностью игнорируются системой обновлений (git). Это означает, что ваши локальные скрипты **никогда не будут затерты или удалены** при обновлении Xilla.

## Как использовать систему плагинов

1. Создайте папку `plugins` в корне вашего репозитория Xilla (рядом с `xilla.py` и папкой `xilla`).
   ```bash
   mkdir -p xilla/plugins
   ```
2. Поместите туда любой `.py` файл, написанный по стандарту модулей Xilla.
   ```bash
   nano xilla/plugins/my_plugin.py
   ```
3. Перезапустите юзербота. Ваш плагин будет автоматически загружен на этапе `register_all`!

## Совместимость с Hikka / Heroku
Начиная с версии B8, ядро Xilla имеет встроенный алиас импортов. Если вы скачали плагин для Hikka (где используются импорты `from hikka import loader`), Xilla **автоматически** перенаправит эти импорты на себя (`xilla`). Вам не нужно вручную редактировать файлы старых модулей!

**Поддерживаются следующие алиасы (Legacy):**
- `hikka` -> `xilla`
- `userbot` -> `xilla`
- `ftg` -> `xilla`
- `heroku` -> `xilla`
