# Создание пакетов (Packages) для Xilla Userbot

*Внимание: Эта функция доступна, начиная с обновления Alpaca B1*

Пакеты — это более сложный, продвинутый уровень расширения Xilla. В отличие от модулей (которые обычно состоят из одного файла `.py` и решают одну задачу), пакеты могут включать в себя полноценные приложения, переписывающие или дополняющие само ядро Xilla.

## Что может пакет?
1. Включать множество взаимосвязанных модулей, которые делят общее состояние.
2. Содержать свои собственные библиотеки (`libs`) и утилиты, которые не будут конфликтовать с ядром.
3. Иметь статические файлы (картинки, данные, `JSON` конфигурации).
4. Заменять дефолтное поведение юзербота (переопределять встроенные команды).

## Структура пакета

Для создания пакета вам понадобится создать репозиторий с определенной структурой:

```text
MyAwesomePackage/
├── manifest.json       # Главный файл с метаданными
├── package_init.py     # Точка входа
├── modules/            # Ваши модули
│   ├── my_advanced_mod.py
│   └── helper_mod.py
├── libs/               # Дополнительные библиотеки
│   └── my_custom_api.py
└── assets/             # Медиа (необязательно)
    └── icon.png
```

## Жизненный цикл пакета

При скачивании пакета встроенный системный модуль `Main` распаковывает архив и помещает пакет в специальную изолированную среду `xilla/packages/`. 
Затем загрузчик ядра Xilla считывает `manifest.json`.

### Пример `manifest.json`

```json
{
  "name": "MyAwesomePackage",
  "version": "1.0.0",
  "author": "YourName",
  "description": "Супер-пакет, который делает всё",
  "entrypoint": "package_init",
  "modules": ["my_advanced_mod", "helper_mod"]
}
```

## Советы по оптимизации
В обновлении **Alpaca B7** мы радикально снизили потребление ОЗУ (вплоть до 500 раз) благодаря агрессивному Garbage Collector-у и удалению StringLoader. 
Пишите свои пакеты с учетом оптимизации:
- Не загружайте в память огромные датасеты без необходимости.
- Используйте асинхронные генераторы (`async for`).
- Не забывайте вызывать `del` для больших объектов или полагайтесь на встроенный в Xilla `memory_optimizer`, который запускается каждые 10 минут.
